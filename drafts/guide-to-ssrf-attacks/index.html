    Guide to SSRF attacks

    An SSRF attack is where an attacker tricks a server into making a request to an arbitrary server. Potential uses:

    • Port scanning and information gathering.
    • Exposing sensitive info from unprotected services (yay microservices).
    • Leaking data from internal metadata services in cloud providers. This can include AWS access keys.

    Anywhere a URL or IP address originates from user-controlled input is a potential attack vector for SSRF.
